# Design Decisions Log

One line per decision made during the audit/upgrade work (audits #1–#9 + S4), with where the full rationale lives. Newest last. When revisiting any of these, update this file.

| # | Decision | Rationale / details | Where |
|---|---|---|---|
| D1 | **Async tasks: Celery optional, thread-pool fallback built-in** | Free-tier deploys must work with no worker; Celery is additive via `render.yaml` uncomment | README "Celery background worker" |
| D2 | **Response caching TTLs**: tutors 5 min, programs 10 min (write-invalidated), analytics 5 min | Staleness documented so it isn't mistaken for a bug | README "API response caching" |
| D3 | **Pagination policy**: growable lists paginate with the DRF envelope; small fixed lists stay arrays; `/api/tutors/` pagination is opt-in via `?limit=` | Changing an existing shape broke 3 pages once — never again without a same-change frontend PR | README + [api-contract tracker](tracker/api-contract-progress.md) AC2 |
| D4 | **API compatibility: additive-only, no `/api/v1` retrofit** | The Capacitor app ships separately; breaking shapes strands old installs. `/api/v2/<resource>` only for unavoidable breaks | README "API compatibility policy" |
| D5 | **Error payload convention**: new/edited custom views return `{"error": "<msg>"}`; DRF defaults untouched; frontend normalises via `getApiError()` | Retrofitting all responses risks the mobile app for zero user benefit | [api-contract tracker](tracker/api-contract-progress.md) AC4 |
| D6 | **`/api/programs/list/` prefix stays as-is** | Renaming is exactly the breaking change D4 forbids; documented instead | [api-contract tracker](tracker/api-contract-progress.md) AC5 |
| D7 | **Single HTTP client**: all internal API calls go through `services/api.js` (auth + refresh interceptors); raw axios only for the refresh call itself, public Cloudinary signature, and external APIs | 101 raw-axios call sites meant token refresh applied to a third of the app | [api-contract tracker](tracker/api-contract-progress.md) AC1 |
| D8 | **500-class errors return generic messages; details go to server logs** (`logger.exception`) | Raw `str(e)` leaked internals; `LOGGING` config makes stdout the source of truth on Render | [error-handling tracker](tracker/error-handling-progress.md) EH3/EH4b |
| D9 | **Payment verification failure ≠ payment failure** — network/5xx on the callback page shows "don't pay again, retry verification" | Prevents double payments after a flaky verify | [error-handling tracker](tracker/error-handling-progress.md) EH4 |
| D10 | **Contrast pass excludes dark surfaces** — `text-slate-400` kept on live-class/whiteboard toolbars and the drawer (detected via same-line dark-bg classes) | slate-400 on dark passes contrast; darkening it would hurt | [accessibility tracker](tracker/accessibility-progress.md) AY-C |
| D11 | **Negative wallet balances are ALLOWED** (2026-07) — intentional admin clawback mechanism; `MinValueValidator` everywhere except `Wallet.balance`; pending withdrawals reserve funds | Recorded as comments at all three debit sites + a test asserting debit-below-zero succeeds | [backend-quality tracker](tracker/backend-quality-progress.md) Q-B |
| D12 | **`backend/scratch/` untracked, not history-purged yet** — purge with `git filter-repo` is a hard precondition of the repo ever going public | Contains DB dumps with real user emails + password hashes | [backend-quality items](items/backend-quality-audit.md) Q1 |
| D13 | **Notifications consolidated on `accounts.Notification`** — the duplicate app model dropped (migration `notifications/0002`); app retained in INSTALLED_APPS only for migration history | Four writers were posting to a model no endpoint read | [backend-quality tracker](tracker/backend-quality-progress.md) Q2/Q-C |
| D14 | **JWT storage (S4)**: refresh token in an httpOnly cookie only; access token in memory; cookie-based bootstrap on page load; legacy body-refresh still accepted server-side | XSS can no longer exfiltrate tokens | [security tracker](tracker/security-audit-progress.md) S4 + README "Authentication" |
| D15 | **Impersonation exception to D14**: the child access token uses a `localStorage('access')` override (survives the hard redirect; parent session stays in the cookie); refresh is disabled while impersonating | Exposure limited to the child's 1-day access token during active impersonation | [security tracker](tracker/security-audit-progress.md) S4 |
| D16 | **Project stage: development, not publishing** — store prep, push notifications, in-app payment flow, and iOS cookie verification are deferred publish-stage work | Explicit so future sessions don't treat them as blockers | README header + [mobile tracker](tracker/mobile-capacitor-progress.md) |
| D17 | **Deployment config is env-driven**: refresh-cookie attributes (`REFRESH_COOKIE_*`), extra CORS origins, Vite proxy target, `PAYSTACK_MOCK_MODE` (default True — flip to False at launch), and admin bootstrap. `create_admin.py` **refuses to run without `ADMIN_PASSWORD`** — it previously reset the admin password to a hardcoded default on every deploy | Removes the two silent production hazards: mocked payments and a publicly known admin password | README env table |
| D18 | **Dev runs same-origin through the Vite proxy** (`/api`, `/media` → :8000; api client uses relative URLs in dev) | Makes the S4 refresh cookie first-party in dev — eliminates browser SameSite/third-party-cookie edge cases on reloads. WebSockets still connect directly (no cookies needed) | `vite.config.js`, `services/api.js` |
